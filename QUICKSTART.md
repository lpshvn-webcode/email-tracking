# 🚀 Быстрый старт

## Отправка письма с измененной датой за 3 шага

### Шаг 1: Установка зависимостей
```bash
cd /Users/rabota/Desktop/email_backdating
pip3 install -r requirements.txt
```

### Шаг 2: Проверка MX сервера (опционально)
```bash
python3 check_mx.py
```

**Ожидаемый результат:**
```
✓ MX записи для домена 'bientotmail.com':
  Приоритет:  10 | Сервер: mail.bientotmail.com
✓ Основной MX сервер: mail.bientotmail.com
```

### Шаг 3: Отправка письма
```bash
python3 send_email_backdated.py
```

**Что произойдет:**
1. Скрипт найдет MX сервер для bientotmail.com
2. Создаст письмо с датой 07.07.2026
3. Прикрепит файл test_attachment.txt
4. Отправит напрямую на mail.bientotmail.com:25

## ⚙️ Изменение параметров

Отредактируйте `config.py`:

```python
SENDER_EMAIL = "ваш_email@domain.com"
RECIPIENT_EMAIL = "получатель@domain.com"
FAKE_DATE = "01.01.2025 12:00:00"  # Формат: DD.MM.YYYY HH:MM:SS
EMAIL_SUBJECT = "Ваша тема"
```

## ✅ Проверка результата

1. **Откройте почтовый ящик** ulnnrpna@bientotmail.com
2. **Найдите письмо** (проверьте папку СПАМ)
3. **Откройте "Показать оригинал"** или "View Source"
4. **Сравните заголовки:**
   - `Date:` → 07.07.2026 (подделанная)
   - `Received:` → 07.01.2026 (реальная)

## ⚠️ Возможные проблемы

### Порт 25 заблокирован?
Многие провайдеры блокируют порт 25. Решения:
- Используйте VPN
- Используйте VPS с открытым портом 25
- Альтернатива: отправка через Gmail (см. README.md)

### Письмо не пришло?
- Проверьте папку СПАМ
- Подождите 1-2 минуты
- Проверьте логи в консоли
