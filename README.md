# Отправка Email с Измененной Датой

## 📚 Дипломная работа по кибербезопасности

Этот проект демонстрирует, как можно изменить заголовок `Date` в email для образовательных и исследовательских целей в области кибербезопасности.

---

## 🎯 Цель эксперимента

Показать разницу между:
- **Заголовок `Date:`** - устанавливается отправителем, может быть подделан
- **Заголовки `Received:`** - добавляются почтовыми серверами, содержат реальное время

| Заголовок | Можно подделать? | Содержит реальное время? |
|-----------|-----------------|--------------------------|
| `Date:` | ✅ Да | ❌ Нет |
| `Received:` | ❌ Нет | ✅ Да |
| `Message-ID:` | ✅ Да | ⚠️ Может содержать timestamp |

## 🛡️ Выводы для дипломной работы

### Что демонстрирует эксперимент:

1. **Легкость подделки визуальной даты**
   - Заголовок `Date` устанавливается клиентом
   - Почтовый клиент показывает эту дату пользователю
   - Не требует специальных навыков для изменения

2. **Невозможность подделки реальных меток времени**
   - Заголовки `Received` добавляются серверами
   - Содержат IP адреса и реальное время
   - Не могут быть изменены отправителем

3. **Важность проверки полных заголовков**
   - При судебной экспертизе анализируются все заголовки
   - Цепочка `Received` показывает реальный путь письма
   - SPF/DKIM/DMARC помогают верифицировать подлинность

### Практическое применение:

- **Фишинг**: Злоумышленники могут использовать старую дату для обхода фильтров
- **Социальная инженерия**: "Письмо было отправлено месяц назад, срочно!"
- **Судебная экспертиза**: Необходимость анализа всех заголовков
- **Корпоративная безопасность**: Обучение сотрудников проверке подлинности

## ⚠️ Возможные проблемы и решения

### Проблема 1: Порт 25 заблокирован

**Симптомы:**
```
✗ Ошибка: Соединение отклонено mx.server.com:25
```

**Решения:**
1. Многие интернет-провайдеры блокируют порт 25
2. Попробуйте использовать VPN
3. Используйте VPS/сервер с открытым портом 25
4. Альтернатива: отправка через Gmail SMTP (см. ниже)

### Проблема 2: Письмо попало в СПАМ

**Причины:**
- Отсутствие SPF/DKIM записей для домена отправителя
- IP адрес в blacklist
- Подозрительный контент

**Решение:**
- Это нормально для тестовых отправок
- Проверьте папку СПАМ у получателя
- Для реальных проектов настройте SPF/DKIM/DMARC

### Проблема 3: dnspython не установлен

**Ошибка:**
```
ModuleNotFoundError: No module named 'dns'
```

**Решение:**
```bash
pip install dnspython
```

## 🔄 Альтернативный метод: Отправка через Gmail

Если прямая отправка не работает, создайте файл `send_via_gmail.py`:

```python
import smtplib
from email.message import EmailMessage
from email.utils import formatdate
from datetime import datetime

# Gmail SMTP
GMAIL_SMTP = "smtp.gmail.com"
GMAIL_PORT = 587
GMAIL_USER = "ваш_email@gmail.com"
GMAIL_PASSWORD = "ваш_app_password"  # Не обычный пароль!

# Создание письма с измененной датой
msg = EmailMessage()
msg['From'] = GMAIL_USER
msg['To'] = "ulnnrpna@bientotmail.com"
msg['Subject'] = "Тест"
msg['Date'] = formatdate(timeval=datetime(2026, 7, 7).timestamp())
msg.set_content("Тестовое письмо")

# Отправка
with smtplib.SMTP(GMAIL_SMTP, GMAIL_PORT) as server:
    server.starttls()
    server.login(GMAIL_USER, GMAIL_PASSWORD)
    server.send_message(msg)
    print("✓ Отправлено через Gmail!")
```

**Настройка Gmail App Password:**
1. Перейдите в настройки Google Account → Security
2. Включите 2FA (двухфакторную аутентификацию)
3. Создайте "App Password" для почты
4. Используйте этот пароль в скрипте
