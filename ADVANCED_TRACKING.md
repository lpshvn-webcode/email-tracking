# 🎯 Advanced Version - Tracking & Image Hosting

## Продвинутая версия с отслеживанием просмотров

Эта версия автоматически:
- ✅ Размещает изображение на сервере
- ✅ Вставляет изображение в HTML письмо
- ✅ Отслеживает открытие письма (tracking pixel)
- ✅ Записывает каждый просмотр изображения
- ✅ Собирает IP, User-Agent, время доступа


## 🌐 Доступ из интернета (ngrok)

Для работы tracking получатель должен иметь доступ к вашему серверу.

### Вариант 1: ngrok (рекомендуется)

```bash
# 1. Установите ngrok
brew install ngrok

# 2. Запустите туннель
ngrok http 5000

# 3. Вы получите публичный URL:
# https://abc123.ngrok.io

# 4. Используйте этот URL в коде
```

**Измените в `web_interface_advanced.py`:**

```python
# Вместо:
image_url = f"http://127.0.0.1:5000/image/{tracking_id}"

# Используйте:
image_url = f"https://abc123.ngrok.io/image/{tracking_id}"
```

### Вариант 2: VPS сервер

Разверните на VPS с публичным IP:

```bash
# На VPS:
python3 web_interface_advanced.py

# Используйте:
image_url = f"http://YOUR_VPS_IP:5000/image/{tracking_id}"
```

## ⚠️ Важные замечания

### Ограничения:

1. **Локальный сервер** - работает только с ngrok/VPS
2. **Порт 25** - может быть заблокирован провайдером
3. **Tracking блокировка** - некоторые почтовые клиенты блокируют изображения
4. **Privacy** - многие клиенты предупреждают о tracking

### Когда tracking НЕ сработает:

- ❌ Если клиент блокирует внешние изображения
- ❌ Если используется VPN/Proxy
- ❌ Если включена защита от tracking (Apple Mail Privacy Protection)
- ❌ Если письмо просматривается в текстовом режиме

### Защита получателей:

**Gmail:**
- Кэширует изображения через свои proxy
- Вы увидите IP Google, а не получателя

**Apple Mail (iOS 15+):**
- Apple Mail Privacy Protection скрывает реальный IP
- Предзагружает контент

**Outlook:**
- Может блокировать внешние изображения по умолчанию

## 🐛 Решение проблем

### Tracking не работает

**Проверьте:**
1. Сервер запущен и доступен из интернета
2. URL в письме правильный (не localhost если шлете вовне)
3. Получатель разрешил загрузку изображений
4. Файл tracking_data/ID.json создается

### Изображение не отображается

**Проверьте:**
1. Файл сохранен в hosted_images/
2. URL доступен в браузере
3. Формат изображения поддерживается
4. Почтовый клиент не блокирует внешние изображения

### JSON пустой или нет данных

**Причины:**
- Письмо еще не открыли
- Почтовый клиент заблокировал tracking
- Используется VPN/защита приватности
